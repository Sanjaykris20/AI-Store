Optimize model parameters to improve the accuracy and relevance of suggestions.

Develop a system to generate real-time product recommendations for individual users.

DATA UNDERSTANDING & USER–ITEM MATRIX (FULL THEORY + EXAMPLES)
What is a Dataset?
Definition (Simple):
A dataset is a collection of information stored in a table-like format.It contains rows and columns, like an Excel sheet.
Columns = features of the data
Example:
UserID
ProductID
Rating
Category
Price
Rows = individual records
Example:One row may represent:
one purchase
one product
one rating
one user action
Example Dataset from Amazon (simplified)
This tells us which user interacted with which product and how much.
Types of Data Used in Recommendation Systems
A recommendation engine uses two major types of data:
A. User Behavior Data
These are the actions performed by the user on the app/website.
1. Clicks
If a user clicks on a product, it means they showed interest.
Example:You click on “Nike Running Shoes” → system learns you like sports shoes.
2. Search History
What users type in the search bar.
Example:User searches “wireless earphones” → recommend audio products.
3. Add to Cart
Strong signal that user likes that item.
Example:User added Boat Airdopes 141 to cart → recommend similar earbuds.

4. Purchase History
Most important behavior because it shows confirmed interest.
Example:If a user bought a Samsung phone, they may like:
Samsung cases
Tempered glass
Samsung smartwatches
5. Ratings & Reviews
Users explicitly tell the system whether they liked something.
Example:User rates a book 5 stars → system suggests similar books.
B. Product Metadata (Item Data)
These are the characteristics of the product.
Examples:
Category: Electronics, Beauty, Clothing
Brand: Apple, Samsung, Nike
Price: ₹499, ₹12999
Color: Black, Blue
Description: “Wireless Bluetooth earphones with mic”
Features: 20-hour battery life, waterproof, noise cancellation
Why product metadata is important?
In content-based filtering, the model recommends:“Items similar to what the user liked earlier.”
To check similarity, the model uses:
category
brand
keywords in description
price range
3. What is Data Preprocessing?
Definition:
Data preprocessing is the step where we clean, fix, and prepare raw data so that the ML model can understand it.
It is like washing vegetables before cooking.Unclean data → bad predictions.
Common Preprocessing Steps
A. Handling Missing Values
Example:Some products may not have a price or category.
We fill missing data, remove it, or replace with averages.
B. Removing Duplicates
Example:If the same user rating is stored twice:
This will confuse the model.
C. Converting Data Types
Example:Price stored as text "₹999" must be converted to number 999.
D. Normalization
Sometimes numbers must be brought to the same range.
Example:
Product price: ₹250–₹50,000
Rating: 1–5
Different ranges → ML strugglesNormalization solves this.
E. Encoding (Converting Text to Numbers)
Models cannot understand text, only numbers.
Example:Category = “Electronics” → convert to number like 0, 1, 2…(One-hot encoding or label encoding)
4. What is a User–Item Interaction Matrix?
Definition:
It is a large table where:
Rows = Users
Columns = Items/Products
Values = how much the user interacted with that item
This matrix is the foundation of collaborative filtering.
Example
How to read this:
U1 rated P1 = 5
U2 bought P4 = 4
U3 never interacted with P3 (value = 0)
What value do we store?
Depending on dataset:
Rating
Click count
Purchase count
1 (clicked) or 0 (not clicked)
5. Sparse Matrix (Very Important Concept)
Definition:
A sparse matrix is a matrix that contains a LOT of zeros.
Why?
Because:
A user interacts with only 5–10 products out of 10,000+ products.
So 99% of the matrix will be zeros.
Example (simple)
Most entries are 0, meaning no interaction.
Why is sparse matrix important?
Collaborative filtering must work on sparse data.
Matrix is huge (thousands × thousands).
A lot of calculations will involve zeros.
This is why recommendation engines use special algorithms to handle it.
6. How Content-Based Filtering Uses Data
Definition:
Content-based filtering recommends products similar to the products the user liked, based on product features.
It uses product-level data:
category
brand
description
price
keywords
Example:
User liked:Samsung Galaxy M14
Model looks for similar products:
Samsung Galaxy M31
Samsung A14
Vivo smartphones in same price range
Phones with similar features
Here, it does NOT care about what other users bought.It cares only about item similarity.
How similarity is calculated?
Simple example:Convert product descriptions into vectors using TF-IDF, then compare using cosine similarity.
But you don't need to teach math to students today — just explain the concept.
7. How Collaborative Filtering Uses Data
Definition:
Collaborative filtering recommends items based on similar users or similar items.
It uses the user–item matrix.

Two types:
A. User–User Collaborative Filtering
“If users A and B have similar taste, recommend to A what B liked.”
Example:
User A:
watches action movies
buys sneakers
loves fitness products
User B:
watches action movies
buys sneakers
→ Recommend fitness products to User B.
B. Item–Item Collaborative Filtering
“If two items are bought together often, recommend them together.”
Example:
Users who bought:iPhone → also bought AirPods.
So if a new user buys an iPhone → system recommends AirPods.
How Semantic search done:
Semantic search , in semantic search we are doing the searching on the basis of the meaning of the words. Not as keyword search where we take 2-3 important words and find the similar words in our document.

 text into vectors.

CONTENT-BASED RECOMMENDATION – STEP BY STEP EXPLANATION
WHAT IS CONTENT-BASED FILTERING?
Content-based filtering is a recommendation method where:
The system recommends products that are similar to the product selected by the user.
It does not depend on other users.It only depends on product information like tags, description, category, etc.
In your case, similarity is calculated using the “Tags” column.
STEP 1: CHECK IF THE PRODUCT EXISTS
First, the system checks whether the product name entered by the user exists in the dataset.
If the product is not found:
It prints a message saying the item is not found.
It returns an empty result.
The process stops there.
If the product exists, then it continues.
STEP 2: CONVERT TEXT (TAGS) INTO NUMBERS
Computers cannot understand text directly.
So the “Tags” column (which contains text like “wireless bluetooth headphones”) is converted into numbers using a method called TF-IDF.
What happens here?
All product tags are taken.
Sentences are broken into individual words.
Common words like “the”, “is”, etc. are removed.
Each word is converted into a numeric value.
A matrix is created where:
Rows = products
Columns = words
Values = importance of that word in that product
Now each product becomes a numeric vector.
STEP 3: CALCULATE SIMILARITY BETWEEN PRODUCTS
After converting text into numbers, the system calculates similarity between all products.
This is done using cosine similarity.
Cosine similarity measures how close two products are based on their tag vectors.
If similarity score is:
Close to 1 → very similar
Close to 0 → not similar
A similarity matrix is created where:Each product is compared with every other product.
STEP 4: FIND THE SELECTED PRODUCT INDEX
The system finds the row number (index) of the product selected by the user.
For example:If the selected product is at row 5,Then index = 5.
This index helps the system know which product to compare with others.
STEP 5: GET SIMILARITY SCORES FOR THAT PRODUCT
Now the system looks at the similarity matrix and extracts similarity scores of the selected product with all other products.
It creates a list like this conceptually:
(Product 0, similarity score)(Product 1, similarity score)(Product 2, similarity score)and so on
STEP 6: SORT PRODUCTS BY SIMILARITY
The system sorts all products based on similarity score in descending order.
Highest similarity comes first.
Important:The first result will always be the product itself,because any product is 100% similar to itself.
STEP 7: SELECT TOP SIMILAR PRODUCTS
After sorting, the system selects the top N similar products.
For example:If top_n = 10,It selects the 10 most similar products.
Usually, the product itself should be removed from the recommendation list.
STEP 8: FETCH PRODUCT DETAILS
Now the system retrieves useful information of the selected similar products, such as:
Product Name
Review Count
Brand
Only these details are returned to the user.
FINAL FLOW IN SIMPLE WORDS
User selects a product.
System checks if product exists.
Product tags are converted into numbers.
Similarity between products is calculated.
Products are sorted by similarity.
Top similar products are selected.
Their details are returned as recommendations.
WHY THIS IS CALLED CONTENT-BASED?
Because recommendation is based on product content (tags),not on other users' behavior.

Abstract Syntax Trees (ASTs) are tree-like representations of the structure of source code. Python provides powerful tools for working with ASTs, allowing developers to analyze, modify, and generate code programmatically.

import ast

# Parse a simple Python expression into an AST
node = ast.parse("x + y")
print(ast.dump(node, indent=2))
1. ast.parse() - The Code-to-Tree Converter
What it does:
Converts text code (string) into a tree structure (AST object) that Python can analyze.
Analogy:
Think of it like Google Translate:
Input: Text in one language (Python code as string)
Output: Structured data (Tree representation)
Detailed Explanation:
node = ast.parse("x + y")
```

**Breaking it down:**

1. **`"x + y"`** - This is a STRING (text), not actual code
   - Python sees it as: `'x + y'` (just characters)
   - It's NOT running `x + y`, just reading it as text

2. **`ast.parse()`** - This function does the magic:
   - Takes the string `"x + y"`
   - Analyzes it like a Python interpreter would
   - Builds a **tree structure** representing the code

3. **`node`** - The result is an AST object (tree):
   - This is NOT a string anymore
   - It's a complex Python object (tree structure)

---
###* *Visual Representation:**
```
INPUT (String):
"x + y"

       ↓ ast.parse() ↓

OUTPUT (Tree Structure):
         Module
            ↓
          Expr
            ↓
         BinOp
        /  |  \
       /   |   \
    Name  Add  Name
    (x)   (+)  (y)
2. ast.dump() - The Tree Viewer
What it does:
Converts the tree structure (AST object) back into a readable text format so humans can see it.
Analogy:
Like taking an X-ray of the tree structure - it shows you what's inside.
Detailed Explanation:
print(ast.dump(node, indent=2))
Breaking it down:
node - The tree structure from ast.parse() 
This is a complex object you can't easily read
It's like a black box
ast.dump() - Shows you what's inside: 
Takes the tree object
Converts it to readable text
Shows the structure in Python-like syntax
Module(
  body=[
    Expr(
      value=BinOp(
        left=Name(id='x', ctx=Load()),
        op=Add(),
        right=Name(id='y', ctx=Load())
      )
    )
  ],
  type_ignores=[]
)
Reading this:
Module - Top-level container (every Python file is a module)
body=[...] - Contents of the module
Expr - An expression (something that returns a value)
BinOp - Binary operation (two things with an operator) 
left=Name(id='x') - Left side is variable x
op=Add() - Operator is +
right=Name(id='y') - Right side is variable y
3. indent=2 - The Formatting Parameter
What it does:
Makes the output pretty and readable by adding indentation (spaces).
Comparison:
WITHOUT indent=2:
print(ast.dump(node))

# Output (all on one line):
Module(body=[Expr(value=BinOp(left=Name(id='x', ctx=Load()), op=Add(), right=Name(id='y', ctx=Load())))], type_ignores=[])
Problem: Hard to read! Everything is squished together.
WITH indent=2:
print(ast.dump(node, indent=2))

# Output (nicely formatted):
Module(
  body=[
    Expr(
      value=BinOp(
        left=Name(id='x', ctx=Load()),
        op=Add(),
        right=Name(id='y', ctx=Load())
      )
    )
  ],
  type_ignores=[]
)
Better! You can see the tree structure clearly.
